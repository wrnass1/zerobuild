# Matching Service

Сервис подбора участников по стеку технологий для идей.

## Эндпоинты

| Метод | Путь              | Описание                                      |
|-------|-------------------|-----------------------------------------------|
| GET   | /match/{idea_id}  | Подбор кандидатов по стеку для указанной идеи |
| POST  | /invite           | Создать приглашение кандидату в идею         |
| POST  | /ideas            | Создать идею (для матчинга)                  |
| POST  | /candidates       | Создать кандидата (для матчинга)             |

## Алгоритм

- Для каждой идеи хранится `required_stack` — список технологий.
- Для каждого кандидата хранится `tech_stack`.
- При запросе `GET /match/{idea_id}`:
  - считается пересечение стеков;
  - коэффициент совпадения = процент технологий идеи, которые есть у кандидата;
  - кандидаты сортируются по коэффициенту (по убыванию).

## Swagger

После запуска сервиса:

- **Swagger UI:** http://localhost:8001/docs  
- **ReDoc:** http://localhost:8001/redoc  

## Запуск (через docker-compose всего проекта)

Из корня репозитория:

```bash
docker-compose up --build
```

Matching Service будет доступен на порту `8001`.
